# CONTRIBUTING

Cảm ơn bạn muốn đóng góp cho AR-Vocab-App! Dưới đây là hướng dẫn nhanh để đóng góp hiệu quả.

1. Trước khi bắt tay:
- Kiểm tra các issue có sẵn.
- Nếu làm feature lớn, mở một issue proposal trước.

2. Quy trình làm việc:
- Fork → tạo branch theo chuẩn:
  - feat/<tên-ngắn>
  - fix/<issue-number>-<tên-ngắn>
- Viết code & test.
- Commit message rõ ràng.
- Gửi Pull Request với mô tả chi tiết và link tới issue (nếu có).

3. Tiêu chuẩn code:
- Tuân theo style guide của ngôn ngữ.
- Thêm test cho logic quan trọng.
- Không commit credentials hay file nhạy cảm.

4. Issues:
- Tạo issue kèm tiêu đề, mô tả, bước tái tạo (nếu bug), thiết bị/OS, ảnh/video/log (nếu có).
- Gắn label phù hợp (bug/enhancement/docs).

5. Review:
- Tôn trọng feedback từ maintainer.
- Giữ PR nhỏ, dễ review.
- Nếu PR lâu không cập nhật, maintainer có thể đóng sau thông báo.

Nếu bạn cần trợ giúp để bắt đầu (ví dụ: muốn một task nhỏ để làm), tạo issue với tag "good first issue" hoặc nhắn cho maintainer.

Cảm ơn bạn!
